Introduction

There are seven primitive data types in JavaScript. Four discussed thus far from Data and Variables are Number, String, Boolean,and Undefined. The other three are Null, BigInt and Symbol. If you wish to learn more about them, check out the MDN page on data types and structures. Aside from Undefined and Null, the other primitive types are wrapped in object wrappers.

You would recognize primitive data types as:


let sampleNumber = 42;
let sampleString = "This is a string";
let sampleBoolean = true;

Objects are a complex data type that contain a collection of properties. Properties are like variables. These properties are also called keys because they are called to access values, functions and nested objects.


let sampleObject = {
    sampleNumber: 42,
    "sample string": "This is a string",
    sampleBoolean: true
}

How to Access an Object's Property


sampleObject.sampleNumber
>> 42

MDN lists two types of properties: data and accessor. Data properties contain, or refer to, values and objects. While methods are functions belonging to, or enacted by, an object, accessor properties entail specific methods, the get and set, used for retrieving and modifying data properties.

Create an Object

There is only one way to declare an object. However, when declaring property names, you can either name them like variables keeping them one word and using lower camel case, or you can name them with strings. It's most important, like variables, not to reuse names among the properties. You can use variable names you have elsewhere in your program because the variables in an object will always be attached to the object. You will see this in the next section.


const objectName = {
    propertyName: value,
    propertyName: value,
    ...
    propertyName: function() {
        statement1;
        statement2;
        ...
        return value;
    }
}

Whether you use let or const to declare an object only affects whether you can reassign a variable to another object. An object's properties can still be modified, defined and deleted even when the object has been declared with const.


const dogObj = {};
dogObj.color = 'red, black and white';
dogObj;
>> Object { color: 'red, black and white' }
dogObj = {};
>> Uncaught TypeError: invalid assignment to const 'dogObj'

When declaring properties, whether you choose to name them like variables or with strings will determine how they can be accessed. Properties also don't have to be declared with the object. You can add properties after the fact.


const pandaObj = {
    ageInYears: 3,
    weightInKilos: 396
};
pandaObj;
>> Object { ageInYears: 3, weightInKilos: 396 }

pandaObj.ageInYears = 5;
pandaObj;
>> Object { ageInYears: 5, weightInKilos: 396 }

Access an Object

There are two ways to access the properties of an object, either dot notation objectName.propertyName or bracket notation objectName[propertyName]. Bracket notation can be used regardless whether you decide to name your properties like variables or with strings. You can even use bracket notation with external variables containing the desired property name. With dot notation, it must be the name of the property and the property's name cannot be a string.

Here's what you can do with dot notation.


const dogObj = {
    name: 'Rover',
    bark() {
        return 'woof woof';
    }
};
dogObj.age = 4;
